  1. Usuario pregunta en CLI

  7. Lo que vino de Internet o ML sube al KB para futuras consultas

QUE SE HIZO EN SESION 3 (busqueda web FORZADA) <<<< ACTUAL
================================================================

PROBLEMA PRINCIPAL: La busqueda en Internet era solo una SUGERENCIA
  - El hook le decia a Claude "usa WebSearch" en texto
  - Claude a veces lo ignoraba y usaba solo ML
  - No habia enforcement real, solo una instruccion en additionalContext

SOLUCION: Busqueda web FORZADA por codigo Python

1. ARCHIVO NUEVO: C:/Hooks_IA/core/web_search.py
   Motor de busqueda web via DuckDuckGo (ddgs library)
   - search_web(query, max_results=5)
   - Retorna: found, results[], summary, internet_pct
   - No necesita API key
   - Calcula cobertura por cantidad/calidad de texto encontrado
   - Import: usa ddgs (nuevo) con fallback a duckduckgo_search (viejo)

2. PAQUETE INSTALADO: ddgs v9.12.1
   pip install ddgs
   (reemplazo de duckduckgo_search que fue renombrado)

3. ARCHIVO REESCRITO: C:/Hooks_IA/hooks/motor_ia_hook.py
   Cambios principales:

   a) NUEVA FUNCION search_internet(query):
      - Importa web_search.search_web()
      - Se ejecuta AUTOMATICAMENTE cuando KB < 80%
      - Log dice "FORCING web search..." (no "suggesting")
      - Retorna internet_content e internet_pct reales

   b) NUEVO FLUJO en main():
      ANTES:
        1. search_kb() -> si KB < 80%, texto dice "usa WebSearch"
      AHORA:
        1. search_kb() -> obtiene kb_content, kb_pct
        2. if kb_pct < 80: search_internet() -> EJECUTA busqueda
        3. build_context() recibe AMBOS resultados reales
        4. Claude recibe datos de KB + Internet ya resueltos

   c) NUEVO TAG en additionalContext: <internet_knowledge>
      - Contiene resultados reales de DuckDuckGo
      - Titulo, snippet y URL de cada resultado
      - Claude los ve como datos, no como instruccion

   d) FIX #2 - FILTRO DE QUERIES BASURA:
      - Nueva funcion is_valid_query()
      - Rechaza: <task-notification>, XML del sistema, queries < 5 chars
      - Antes estos queries iban a NotebookLM y causaban errores HTTP 400

   e) FIX #3 - MANEJO DE UNICODE:
      - Nueva funcion sanitize_text()
      - Elimina surrogate characters (\udc8f etc) que crasheaban
      - Se aplica a input Y a query antes de procesar
      - Antes: UnicodeEncodeError crasheaba todo el hook

4. build_context() ACTUALIZADO:
   - Recibe 5 params: query, kb_content, kb_pct, internet_content, internet_pct
   - ML se calcula como: max(0, 100 - kb_pct - internet_pct)
   - 4 escenarios de instrucciones:
     * KB >= 80%: "usa KB como base"
     * KB > 0 + Internet > 0: "KB parcial, complementado con Internet"
     * KB = 0 + Internet > 0: "sin KB, usa Internet como base"
     * Nada: "usa ML"

5. save_state() ACTUALIZADO:
   - Ahora guarda internet_pct ademas de kb_pct
   - needs_save basado en (kb_pct + internet_pct) < 100
