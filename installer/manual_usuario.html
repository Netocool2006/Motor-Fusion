1. Introduccion

Motor Fusion IA es un sistema de inteligencia artificial adaptativa que aprende automaticamente de cada sesion de trabajo del usuario. A diferencia de herramientas convencionales, Motor Fusion no requiere configuracion manual de reglas ni entrenamiento explicito: observa como trabajas, detecta patrones, registra soluciones a problemas y construye una base de conocimiento personalizada que mejora con el tiempo.

El motor fue disenado para profesionales que trabajan con multiples sistemas (SAP CRM, Monday.com, herramientas de propuestas, BoM, SOW, entre otros) y necesitan que su asistente de IA recuerde contexto entre sesiones, evite repetir errores y sugiera soluciones basadas en experiencia previa.

Que aprende Motor Fusion IA

  • Patrones de trabajo: secuencias de acciones que el usuario repite frecuentemente.
  • Errores y soluciones: cuando un problema se resuelve, el motor registra la causa y la correccion para evitar repetirlo.
  • Hechos del dominio: datos especificos de cada sistema (codigos SAP, estructuras de BoM, reglas de negocio).
  • Preferencias del usuario: formato de salida, idioma, nivel de detalle preferido.
Nota Motor Fusion IA funciona de forma completamente local. Tus datos nunca salen de tu computadora a menos que configures explicitamente la sincronizacion con GitHub.

Haz doble clic en install.bat. Se abrira una ventana de terminal con el asistente de instalacion.

4. Como Funciona

Motor Fusion IA opera como una capa inteligente entre el usuario y sus herramientas CLI. Su arquitectura se compone de cinco modulos principales que trabajan en conjunto:

Arquitectura General

+----------+ +-----------+ +----------+ +------------------+ | | | | | | | | | Usuario | ---> | CLI Tool | ---> | Hooks | ---> | Motor Fusion IA | | | | (Claude, | | (pre/ | | | | | | Gemini, | | post | +--------+---------+ | | | Ollama) | | session)| | +----------+ +-----------+ +----------+ +------+------+ | | +----+----+ +----+----+ |Knowledge| |Learning | | Base | | Memory | |(dominios)| |(sesion) | +---------+ +---------+

Aprendizaje Automatico

El motor aprende de cada sesion sin intervencion del usuario. Cuando inicias una sesion con cualquier CLI soportado, los hooks se activan automaticamente:

  • Hook pre-sesion: carga contexto relevante de la Knowledge Base y lo inyecta como contexto al LLM.
  • Hook post-sesion: analiza la sesion completada, extrae patrones, errores resueltos y hechos nuevos, y los almacena.

Knowledge Base (Base de Conocimiento)

La KB organiza todo el conocimiento en dominios tematicos:

DominioContenido tipico
sap_tierraPatrones de SAP CRM, codigos de transaccion, estructuras de oportunidad
sowEstructura de Statements of Work, clausulas, formatos
bomBills of Materials, referencias de productos, precios
mondayFlujos de Monday.com, automatizaciones, campos personalizados
business_rulesReglas de negocio, politicas internas, validaciones
catalogCatalogo de productos y servicios
generalTodo lo que no encaja en otro dominio

Tipos de Conocimiento

  • Patrones: secuencias de acciones que se repiten y funcionan.
  • Hechos (Facts): datos concretos validados (ej: "El codigo de producto X es 12345").
  • Errores resueltos: un error, su causa raiz y la solucion aplicada.
  • Preferencias: como el usuario prefiere que se presenten los resultados.

5. Uso Diario

Lo mas importante No necesitas hacer nada especial. El motor aprende solo con cada sesion de trabajo. Simplemente usa tu CLI habitual (Claude Code, Gemini, Ollama) y Motor Fusion IA hara el resto en segundo plano.

Ver que ha aprendido el motor

Para exportar un resumen de todo lo aprendido:

python learning_memory.py export

Esto genera un archivo con todos los patrones, hechos y errores resueltos organizados por dominio.

Buscar en la Knowledge Base

Para buscar conocimiento especifico por dominio y palabra clave:

# Buscar en el dominio SAP
python knowledge_base.py search --domain sap --query "agregar items"

# Buscar en todos los dominios
python knowledge_base.py search --query "precio"

# Buscar errores resueltos
python knowledge_base.py search --domain general --query "unicode"

Ver estadisticas

Para ver cuantos patrones, hechos y errores tiene registrados:

python knowledge_base.py stats

Ejemplo de salida:

Knowledge Base - Estadisticas
=============================
Dominio          Patrones  Hechos  Errores
sap_tierra            12       8        5
sow                    4       3        1
bom                    6      11        2
monday                 3       2        0
general                8       5        4
-----------------------------
TOTAL                 33      29       12

Flujo de trabajo tipico

  1. Abres Claude Code (o tu CLI preferido) y trabajas normalmente.
  2. El hook pre-sesion carga contexto relevante automaticamente.
  3. Durante la sesion, si se resuelve un error o se descubre un patron, el motor lo registra.
  4. Al cerrar la sesion, el hook post-sesion extrae y almacena aprendizajes.
  5. En la siguiente sesion, el motor ya tiene ese conocimiento disponible.

6. Respaldo y Sincronizacion (GitHub Sync)

Motor Fusion IA puede sincronizar tu Knowledge Base con un repositorio privado de GitHub, permitiendote hacer respaldo y restaurar en otra computadora.

Sincronizar a GitHub

python sync_to_github.py

La primera vez te pedira el URL de tu repositorio privado y configurara las credenciales. Las sincronizaciones siguientes son automaticas.

Restaurar en una nueva computadora

python restore_from_github.py --repo https://github.com/tu-usuario/tu-repo-privado.git

Esto descargara toda tu Knowledge Base y la instalara en la nueva maquina.

Que se sincroniza y que no

Se sincronizaNO se sincroniza
Knowledge Base (patrones, hechos, errores)Credenciales y tokens
Learning Memory (historial de aprendizaje)Archivos temporales de sesion
Configuracion de dominiosCache local
Preferencias de usuarioLogs de depuracion
Seguridad Usa siempre un repositorio privado de GitHub. Nunca sincronices tu Knowledge Base en un repositorio publico, ya que puede contener datos sensibles de tus clientes y proyectos.

7. Adaptadores CLI

Motor Fusion IA se conecta a diferentes herramientas CLI mediante adaptadores. Cada adaptador sabe como inyectar contexto y extraer aprendizajes del CLI correspondiente.

Claude Code CLI

Es el adaptador principal y el mas completo. Se configura automaticamente durante la instalacion mediante hooks en settings.json: 

// ~/.claude/settings.json (configurado automaticamente)
{
  "hooks": {
    "PreToolUse": [...],
    "PostToolUse": [...],
    "PreSession": [...],
    "PostSession": [...]
  }
}
  • Inyeccion de contexto de KB al inicio de cada sesion.
  • Captura automatica de patrones y errores.
  • Soporte completo para todos los dominios.

Gemini CLI

Adaptador preparado como stub para integracion futura con Google Gemini CLI. La estructura esta lista; solo falta que Gemini CLI soporte un sistema de hooks similar.

  • Estructura de adaptador lista.
  • Interfaz de inyeccion definida.
  • Pendiente: implementar cuando Gemini CLI soporte hooks.

Ollama (LLM local)

Permite usar modelos de lenguaje locales (Llama, Mistral, Phi, etc.) con inyeccion de Knowledge Base. Ideal para trabajo sin conexion a internet. 

# Iniciar chat con Ollama e inyeccion de KB
python ollama_adapter.py chat --model llama3 --domain sap
  • Inyecta contexto relevante de la KB en cada prompt.
  • Funciona completamente offline.
  • Soporta cualquier modelo disponible en Ollama.

8. Desinstalacion

Para desinstalar Motor Fusion IA, ejecuta:

python uninstall.py

Que se elimina

  • Hooks configurados en settings.json
  • Scripts del motor en el directorio de instalacion
  • Archivos temporales y cache

Que se preserva

  • Tu Knowledge Base completa (~/.adaptive_cli/)
  • Learning Memory con todo el historial de aprendizaje
  • Respaldos en GitHub (si configuraste sincronizacion)
Nota La desinstalacion preserva tus datos por defecto. Si deseas eliminar tambien la Knowledge Base, agrega la bandera --purge:

python uninstall.py --purge

9. Solucion de Problemas (FAQ)

"Los hooks no disparan"
Verifica que el archivo settings.json de tu CLI contenga la configuracion de hooks.

Para Claude Code: 
# Verifica que exista la configuracion
cat ~/.claude/settings.json | python -m json.tool
Si no hay seccion "hooks", vuelve a ejecutar el instalador o agrega la configuracion manualmente.
"Error de permisos al ejecutar scripts"
Windows: Ejecuta la terminal como Administrador (clic derecho > "Ejecutar como administrador").

macOS/Linux: Verifica los permisos del directorio: 
chmod -R 755 ~/.adaptive_cli/
chmod +x *.py *.sh
"No encuentra patrones en la Knowledge Base"
Verifica que el directorio de datos existe y contiene archivos: 
# Verificar que el directorio existe
ls -la ~/.adaptive_cli/

# Verificar la KB
python knowledge_base.py stats
Si el directorio no existe, la instalacion puede haber fallado. Ejecuta el instalador nuevamente.
"Errores de Unicode en Windows"
Motor Fusion IA incluye un fix automatico para la consola de Windows (_fix_windows_stdout) que maneja la codificacion cp1252. Si aun asi ves caracteres raros: 
# Forzar UTF-8 en la terminal
chcp 65001

# O configurar la variable de entorno
set PYTHONIOENCODING=utf-8
Este problema es comun en terminales de Windows con codepage 1252 y no afecta el funcionamiento del motor.
"El instalador falla al detectar Python"
Asegurate de que Python 3.10+ este en el PATH del sistema: 
python --version
# o
python3 --version
En Windows, el instalador puede usar su Python embebido. Si falla, instala Python desde python.org y marca la opcion "Add to PATH" durante la instalacion.
"La sincronizacion con GitHub falla"
Verifica que:
  1. Tienes git instalado y configurado.
  2. El repositorio remoto existe y es accesible.
  3. Tus credenciales de GitHub estan configuradas (token o SSH key).
git config --global user.name
git config --global user.email
git ls-remote https://github.com/tu-usuario/tu-repo.git

10. Glosario

Motor IA
El nucleo del sistema. Coordina el aprendizaje automatico, la gestion de conocimiento y la inyeccion de contexto en cada sesion CLI.
Knowledge Base (KB)
Base de conocimiento persistente organizada por dominios tematicos. Almacena patrones, hechos y errores resueltos en formato estructurado.
Learning Memory
Componente que registra el historial de aprendizaje sesion por sesion, incluyendo que se aprendio, cuando y en que contexto.
Hook
Fragmento de codigo que se ejecuta automaticamente antes o despues de una accion del CLI (inicio de sesion, uso de herramienta, fin de sesion).
Adaptador (Adapter)
Modulo que conecta Motor Fusion IA con un CLI especifico (Claude Code, Gemini, Ollama), traduciendo formatos y protocolos.
Patron (Pattern)
Secuencia de acciones que el motor identifica como recurrente y exitosa. Se almacena para reutilizacion en sesiones futuras.
Hecho (Fact)
Dato concreto y verificado extraido de una sesion de trabajo. Ejemplos: codigos de producto, nombres de campos, URLs de sistemas.
Dominio (Domain)
Categoria tematica que organiza el conocimiento en la KB. Ejemplos: sap_tierra, sow, bom, monday, general.
Sesion (Session)
Una interaccion completa del usuario con un CLI, desde el inicio hasta el cierre. El motor extrae aprendizajes al finalizar cada sesion.
Inyeccion de Contexto
Proceso mediante el cual el motor agrega conocimiento relevante de la KB al prompt del LLM antes de que este genere una respuesta.
Sincronizacion (Sync)
Proceso de respaldo de la Knowledge Base a un repositorio privado de GitHub para persistencia y portabilidad entre computadoras.
Error Resuelto
Registro que contiene un error encontrado, su causa raiz y la solucion aplicada. Permite al motor evitar repetir el mismo error.
